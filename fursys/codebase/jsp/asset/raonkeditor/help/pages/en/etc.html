Hangul Input Problem(Firefox 41.0 version)

Phenomenon

In the Firefox 41.0 version from the k editor, When you enter the Hangul text after changing the line, the curser moves to the previous line. After certain amount time has passed, the cursor is moved forcibly to the first row. This phenomenon can occur in any HTML.

Cause

In Firefox 41.0 version, The cause of the error of Hangul input function is from the collision of the TSF(Text Service Framework is new IME API.) function of MS that has been provided since Windows Vista.

Solution

Please upgrade to the Firefox 42.0 version. (No automatic upgrade support)

Special Characters, Chinese character input problem(Chrome 77.0.3865.90 version)

Phenomenon

In Chrome 77.0.3865.90 and later, special characters and Chinese characters disappear when entering Korean characters after in the K Editor.

Cause

Chrome browser itself bug

Solution

Please upgrade to the Chrome newest version(81.0.4044.113Later versions)(automatic upgrade support)
